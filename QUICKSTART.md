# クイックスタートガイド



初心者の方向けに、最短でキャリアアンカー診断アプリを作る手順をまとめました!



## ステップ1: kintoneアプリを作る（10分）



### 1-1. アプリを新規作成

1. kintoneにログイン

2. 右上の「+」→「アプリを作成」

3. 「はじめから作成」を選択

4. アプリ名: **キャリアアンカー診断**



### 1-2. フィールドを追加



#### 基本情報（2個）

| フィールド名 | タイプ | フィールドコード |

|------------|--------|-----------------|

| 氏名 | 文字列（1行） | name |

| 診断日 | 日付 | diagnosis_date |



#### 質問セクション（Q1〜Q40）



**重要:** 以下を40回繰り返します!



1. 「スペース」フィールドを追加

   - 要素ID: `question_1` (2つ目は `question_2`、以降同様)



2. 「ラジオボタン」フィールドを追加

   - フィールド名: `Q1` (2つ目は `Q2`、以降同様)

   - フィールドコード: `q1` (2つ目は `q2`、以降同様)

   - 選択肢: `1`, `2`, `3`, `4`, `5`（改行で5つ）

   - 並び: 横に並べる

   - 必須: チェックしない



**繰り返しポイント:**

- Q1〜Q40まで、スペース→ラジオボタンのペアを40組作る

- スペースの要素ID: question_1, question_2, ..., question_40

- ラジオボタンのコード: q1, q2, ..., q40



#### お気に入り選択セクション



1. 「スペース」フィールドを追加

   - 要素ID: `favorites_container`



2. 「ドロップダウン」フィールドを3つ追加

   - 1つ目: フィールドコード `favorite_1`

   - 2つ目: フィールドコード `favorite_2`

   - 3つ目: フィールドコード `favorite_3`

   - 選択肢: 「選択してください」などのダミーを1件登録（JavaScriptが候補を上書き）

   - 初期値: 作成したダミー選択肢

   - 必須: チェックしない



> ℹ️ ドロップダウンは最低1件の選択肢が必要です。ダミー以外は登録せず、候補の制御は JavaScript に任せます。

>

> ✅ 40問回答後は最高点の質問から必ず3つ選びます。最高点の設問が3つ未満の場合はその設問が自動的に必須となり、不足分だけ次点のスコアから選びます（同じ質問は選べません）。



#### スコアセクション（8個）



| フィールド名 | タイプ | フィールドコード | 初期値 |

|------------|--------|-----------------|-------|

| TFスコア | 数値 | score_tf | 0 |

| GMスコア | 数値 | score_gm | 0 |

| AUスコア | 数値 | score_au | 0 |

| SEスコア | 数値 | score_se | 0 |

| ECスコア | 数値 | score_ec | 0 |

| SVスコア | 数値 | score_sv | 0 |

| CHスコア | 数値 | score_ch | 0 |

| LSスコア | 数値 | score_ls | 0 |



**すべて「表示のみ」にチェック**



#### 判定結果



| フィールド名 | タイプ | フィールドコード |

|------------|--------|-----------------|

| あなたのキャリアタイプ | 文字列（複数行） | career_type_result |



**「表示のみ」にチェック**



### 1-3. 保存

画面右上の「設定完了」→「アプリを更新」

## ステップ2: JavaScriptをアップロード（5分）

### 2-1. kintone UI Component v1（KUC）を追加
1. [公式リリースページ](https://github.com/kintone-labs/kintone-ui-component/releases) から kuc.min.js（または kintone-ui-component.min.js）を取得。v1 は単一の JS にスタイルが内包されているため、CSS ファイルはありません。
2. プロジェクト内（例: libs/kuc/）に保存し、後でアップロードできるようにしておく。

> ✅ KUC を読み込まないと、お気に入りのドロップダウンがダミーのままで動きません。

### 2-2. カスタマイズ画面を開く
1. アプリの設定（歯車アイコン）
2. 「設定」タブ
3. 「JavaScript / CSSでカスタマイズ」

### 2-3. ファイルをアップロード
1. 「PC」タブを選択
2. CSS欄: src/style.css のみ追加（KUC は JS にスタイルが含まれるため不要）
3. JavaScript欄: kuc.min.js → src/career-anchor.js の順で追加
4. 「保存」ボタン

### 2-4. アプリを更新
画面右上の「アプリを更新」→「更新」

## ステップ3: テスト（5分）



### 3-1. レコードを作成

1. アプリを開く

2. 「+」ボタンで新規レコード

3. 氏名を入力



### 3-2. 診断してみる

1. Q1〜Q40に回答（適当に1〜5を選ぶ）

2. 全て回答すると「お気に入り選択」が表示される

3. 最高点の質問をすべて選び、不足分を次点の質問から選んで合計3つにする（同じ質問は選べません）

4. 「保存」ボタン



### 3-3. 結果を確認

- 8つのスコアが表示される

- 「あなたのキャリアタイプ」に結果が表示される

## うまくいかない時は?



### エラーが出る場合

1. **F12キー**を押してコンソールを開く

2. 赤いエラーメッセージを確認

3. フィールドコードのスペルミスが多いです!



### よくあるミス



#### ケース1: 質問文が表示されない

→ スペースフィールドの要素IDを確認

- ❌ `Question_1` や `question1`

- ✅ `question_1`



#### ケース2: お気に入りが表示されない

→ 以下を確認

- 40問すべて回答したか?

- スペースの要素IDが `favorites_container` か?



#### ケース3: スコアが0のまま

→ フィールドコードを確認

- ラジオボタン: `q1`〜`q40`

- 数値フィールド: `score_tf`, `score_gm` など



#### ケース4: お気に入りが保存できない

→ 以下を確認

- 最高点の質問をすべて選んだか?

- 3つとも選択されているか?

- 同じ質問を重複選択していないか?
