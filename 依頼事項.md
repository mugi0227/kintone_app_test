# 目的
1. js,css,kintone,formbridge,kviewerに詳しくない人間が自分で本アプリケーションをkintone,formbridge,kviewerに実装できること
2. js,cssに詳しくない人間が、わかりやすくその動作の仕組みを理解でき、かつ毒性（インジェクション）がないことを確信できること
3. 本アプリケーションがkintoneなどのアップデートなどにより正常な動作をしなくなった場合に、js,cssに詳しくない人間が、自分で修正できること

# 依頼事項
目的の1~3を達成するために、ドキュメントを生成してください。
できれば、2.に関しては視覚的にもわかりやすいようにできたらうれしい。
1,3はどちらかというとアプリケーションを運用する現場向け、2はkintoneを管理する弊社事務局向け。
事務局懸念事項として、運用時に問題が発生するリスクがあり、DOM操作に関して印象がよくない。このアプリでのDOM操作に関して、kintoneなど各種アップデートによって破壊されないか、破壊された場合どうすればいいかがわかるような説明にしてください。

# 注意事項
①実装マニュアル→ラジオボタンの選択肢は1-5
- 文字列（複数行） `career_type_result`
- 日時 `diagnosis_date`
も必要
Formbridgeでは、氏名と日時以外はすべて非表示に。- 文字列（複数行） `career_type_result`も必要。
kviewerでも- 文字列（複数行） `career_type_result`も必要。作成日時ではなくdiagnosis_date

②アーキテクチャ
これは、jsやcssの中身すべてについて、コードの各箇所が何をしているのかということがわかるようなドキュメントとしてください。ブラックボックス化を解消して、安全であることを「証明」できないと意味がないです。

③メンテナンス
具体的に、AIチャットを用いてどうやって改修するかを記述してください。
